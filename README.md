# ProteoLab Suite 🧬

**ProteoLab Suite** is an interactive R Shiny application designed to automate quality control and data analysis for quantitative proteomics, specifically focusing on **TMT (Tandem Mass Tag)** workflows processed with **FragPipe**.

The suite generates dynamic HTML reports to ensure optimal labeling efficiency and sample mixing before proceeding with downstream LC-MS/MS analysis.

---

## 🚀 Main Modules

### 1. Ratio Check
This module verifies if TMT-labeled samples are properly balanced. It calculates current ratios and determines the exact volumes required to correct discrepancies.

* **Required Inputs:**
    * `psm.tsv`: The Peptide-Spectrum Match file generated by **FragPipe**.
    * `annotation.txt`: The channel annotation file used in **TMT-Integrator**.
* **Configurable Parameters:**
    * **Label Type:** Supports TMT (10/11-plex) and TMTpro (16/18-plex).
    * **LysN Extra Labeling:** Adjusts modification logic if LysN was utilized.
    * **Desired Volume:** The target final volume ($\mu L$) for the pool.
* **Report Insights:**
    * Distribution of missed cleavages and charge states ($z$).
    * Labeling efficiency at N-termini and Lysine (K) residues.
    * **Mixing Table:** Precise calculation of how much volume to pipette from each channel.
    * **ACN Adjustment:** Calculation of Formic Acid needed to reduce ACN concentration from 26.6% to 3% for column loading.

### 2. Overlap Analysis
Ideal for fractionated experiments, this module evaluates the redundancy or complementarity between the analyzed fractions.

* **Required Inputs:** FragPipe's `psm.tsv` file.
* **Key Metrics:**
    * Unique peptide and protein counts per fraction.
    * **Jaccard Index:** Heatmaps showing the percentage of peptide and protein overlap between all fractions (lower triangle matrix).

## 🛠️ System Requirements

To run the application locally, you need **R** and the following packages installed:

```r
install.packages(c("shiny", "rmarkdown", "dplyr", "tidyr", "stringr", 
                   "ggplot2", "kableExtra", "janitor", "ggpubr", "readr"))
```

## 📖 How to Use

1. Ensure the report templates (`ratio_check_report.Rmd` and `fraction_overlap_report.Rmd`) are in the same directory as the application file (`app.R`).

2. Launch the app in RStudio by clicking the "Run App" button or using `shiny::runApp()`.

3. Ratio Check Tab:

    * Upload your PSM and Annotation files.

    * Select the TMT type and set the target volume.

    * Click "Generate Report" and then "Download Report" to save the HTML output.

4. Overlap Tab:

    * Upload the PSM file to visualize fractionation efficiency and identify potential cross-contamination between fractions.
